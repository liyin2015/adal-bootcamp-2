2 minutes to start

Install & Launch

# 1. Get iTerm2 (recommended terminal)
$ brew install --cask iterm2
 
# 2. Install AdaL
$ curl -fsSL https://adal.sylph.ai/install.sh | bash
 
# 3. Run in tmux (packaged — don't worry about it)
$ tmux new -s adal
 
# 4. Launch
$ adal --mode engineer # full loop engineering
$ adal # single worker

Full guide: docs.sylph.ai/getting-started/quickstart

Two Ways to Start

# Engineer mode

Full loop engineering.
Hides complexity:
• worker setup
• model selection
• harness wiring
• self-adapts to you

You drive. AdaL engineers.

# Worker agent
$ adal
 
Single worker mode.
Direct interaction:
• prompt directly
• no hidden orchestration
• you control everything

You prompt. Worker executes.

Cost-Effective Model Routing

Model Recommendations

AdaL engineer mode picks models per task. Here's what we recommend — cheaper first, upgrade after compaction.

💰 Budget tier (default)

  • Planner / Engineer: GLM-5.2 (no visual)
  • Browser use: MiniMax M3 > Gemini 3.5 Flash

Runs the full loop cheaply. Good for exploration & iteration.

🔥 Premium tier (post-compaction)

  • Engineer: Opus 4.6 (with visual)
  • Browser use: GPT-5.5 (super expensive)

Switch after /compact when context is distilled — fewer tokens, higher quality.

You can't burn tier-1 tokens in a loop. Start cheap, upgrade on compaction.
